# Privacy Boundaries

Updated: 2026-04-29

Woong Monitor Stack measures usage metadata. It must never become a hidden
surveillance, input capture, or screen capture tool.

## Explicitly Forbidden

- Keylogging.
- Recording typed characters.
- Recording passwords.
- Reading form inputs.
- Clipboard capture.
- Screen recording.
- Periodic user activity screenshots.
- Private message or content scraping.
- Browser page content scraping.
- Covert tracking without visible status.
- Android touch coordinate tracking across other apps.
- Android text input tracking across other apps.
- Android Accessibility Service monitoring unless explicitly approved in a
  future PRD update.

## Allowed Metadata

- Foreground app/process/window metadata.
- Window title only if user settings allow it.
- Active/idle state based on last input timestamp.
- Browser domain or full URL only under documented privacy settings.
- Android app usage duration through UsageStatsManager and user-granted Usage
  Access.
- Local test screenshots of this app's own dashboard UI only.

## Screenshot Boundary

WPF and Android screenshot tools are for testing this app's own UI. They must
not capture the user's desktop activity, other apps, browser contents, or other
Android apps as product telemetry.

Screenshots generated by local acceptance tools are developer artifacts, not
usage data. They must stay under ignored artifact folders.

The WPF local snapshot tool is guarded by architecture tests that require
captures to be taken from the target app window or UI elements, not from the
desktop or arbitrary screen regions.

## Consent And Visibility

- Collection must be visible in the app UI.
- Permissions must be explicit.
- Android periodic collection scheduling requires both Usage Access permission
  and a persisted collection-enabled setting.
- Sync must be opt-in.
- Android sync defaults to off and workers must suppress uploads until the
  persisted opt-in setting is enabled.
- Android app backup is disabled for the MVP app so local Room usage metadata
  is not copied into device backup flows.
- Browser URL capture must be opt-in beyond generic browser app duration.
- Full URL capture must be separate from domain-only capture.
- Raw events need retention limits and must avoid sensitive content.

## Automated Guardrails

Architecture tests now scan product source and manifests for forbidden tracking
capabilities. The guardrails fail the build if product code introduces keyboard
hooks, raw keyboard state APIs, clipboard capture APIs, screen capture APIs,
Android Accessibility-service monitoring, Android text-input hooks, Chrome page
content scripting, browser history/cookie permissions, or tab/desktop capture.

Android manifest tests allow metadata-oriented permissions such as Usage Access
and notifications, but reject invasive monitoring permissions and Accessibility
services.

## Metadata Reminder

This project answers:

> Which apps/sites did I spend time in, and for how long?

It does not answer:

> What did the user type, read, see, copy, say, or send?
